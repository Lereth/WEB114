Questions

Why is textContent safer than innerHTML?
Using textContent helps in reducing the risk of secuirtiy vulnerablities such as Cross-site Scripting. When you use innerHTML, you're allowing HTML content to be inserted directly, and if that content comes from an untrusted source, it could include malicious scripts.

How are HTML elements connected to JavaScript variables using DOM selection?
HTML elements are connected using methods like getElementById(), which selects elements by their ID, class, or tag.

What would happen to the text box with id="text" if I added this line to the code above it: text.textContent = "Hello!"
The text inside the element with id="text" would change to display "Hello!".
